## TL;DR

Can robot manipulation data be collected in-the-wild without robots? UMI uses $371 hand-held grippers + GoPro to collect human demonstrations, combining Diffusion Policy with latency matching and relative trajectory representations to learn policies deployable zero-shot across multiple robot platforms.

## Problem

Existing robot learning data collection faces two key limitations. First, **teleoperation requires robots**, confining data collection to labs and making it expensive. Second, lab-collected data produces policies that **fail to generalize to unseen environments**. Complex tasks like dynamic tossing or bimanual coordination are often impossible to demonstrate via teleoperation entirely.

## Key Idea

UMI's core is a dual design of **Demonstration Interface** and **Policy Interface**:

**Demonstration Interface** — humans use hand-held grippers in the wild:
- **155° Fisheye GoPro**: The only sensor, preserving center resolution while capturing peripheral information
- **Side Mirrors**: Physical mirrors in the fisheye periphery providing **implicit stereo depth** information
- **IMU-aware SLAM**: GoPro built-in IMU + ORB-SLAM3 for 6-DOF tracking (mean error 6.1mm, 3.5°)

**Policy Interface** — deploying collected data to robots:
- **Latency Matching**: Time synchronization across observation streams (camera, pose, gripper) with execution delay compensation
- **Relative Trajectory Action**: SE(3) transforms relative to current EE pose, more robust to tracking errors and camera displacements than delta or absolute representations

<Collapsible title="Detailed Methods">

### Relative Trajectory vs Delta vs Absolute Action

- **Relative Trajectory (UMI)**: Entire action sequence relative to EE pose at inference start. Robust against tracking errors and camera displacements
- **Delta**: Each action relative to immediately previous → error accumulates
- **Absolute**: Requires global coordinate frame → infeasible for in-the-wild data

### Fisheye vs Pinhole Rectification

Rectifying a 155° FoV fisheye image to pinhole model severely stretches peripheral views while compressing central information. UMI uses raw fisheye images as direct policy input.

### Side Mirror Digital Reflection

Digitally reflecting (flipping) mirror crops makes mirror view orientations consistent with main camera, improving policy learning. Without reflection: 85%, with reflection: 100% success rate.

### Latency Matching

1. Physically measure latency for each observation stream (RGB, EE pose, gripper width)
2. Align to stream with highest latency (usually camera)
3. Send action commands ahead of time to compensate for execution latency

</Collapsible>

## Key Results

- **Cup Arrangement**: 20/20 (**100%**), cross-robot (Franka FR2) 18/20 (90%)
- **Dynamic Tossing**: 105/120 (**87.5%**) — without latency matching: 57.5%
- **Bimanual Cloth Folding**: 14/20 (**70%**) — without inter-gripper proprioception: 30%
- **Dish Washing**: 14/20 (**70%**) — long-horizon (multiple rigid/deformable/articulated objects)
- **In-the-wild Generalization**: 1,400 demos from 30 locations → 71.7% success in unseen environments (narrow-domain data: 0%)
- **Data Collection Speed**: **3× faster** than teleoperation; dynamic tossing impossible with teleoperation

## Achievements and Limitations

**Key achievements**
- **$371 hand-held gripper** enables in-the-wild data collection without robots — removes teleoperation's environment/cost constraints
- Enables demonstration and learning of **tasks impossible via teleoperation** (dynamic tossing, bimanual coordination)
- Relative trajectory + latency matching enables **zero-shot deployment across multiple robot platforms** (UR5e, Franka FR2)

**Limitations**
- Cannot know downstream robot kinematics at collection time, **relying on data filtering** — hardware-aware policy learning needed
- Visual SLAM **fails in texture-poor environments** (pure white walls)
- UMI gripper (780g) is heavier and bulkier than bare hands — **demonstration efficiency is 48-64%** of bare-hand speed
- 🤖 Supports only 2-DOF grippers, cannot leverage rich dexterous manipulation; partially addressed by follow-up DexUMI
