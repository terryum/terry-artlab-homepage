## TL;DR

Can the "experiment → check results → revise idea → repeat" loop that ML researchers have done manually for 20 years be delegated wholesale to AI agents? Karpathy's autoresearch makes this real. An AI agent autonomously modifies train.py, runs experiments, and evaluates results — in 2 days it ran ~700 experiments, reducing GPT-2 training time by 11% and discovering real bugs that a veteran researcher had missed.

## Problem

The core loop of ML research is inherently iterative: come up with an idea, modify code, run an experiment, check validation loss, come up with new ideas based on results. Karpathy has done this manually for 20 years. The problem is that **the human is the bottleneck** — waiting while experiments run, testing only one hypothesis at a time, needing sleep. GPUs are fast enough; the human judgment loop can't keep up.

## Key Idea

autoresearch's design is radically simple:

- **3 files**: `prepare.py` (fixed constants/data), `train.py` (the only file the agent modifies), `program.md` (human-written research instructions)
- **Fixed 5-minute time budget**: Every experiment runs for the same wall clock time, enabling fair comparison regardless of architecture/batch size/model size changes
- **Single metric**: `val_bpb` (validation bits per byte) — lower is better, vocabulary-size independent
- **Agent's role**: Review the sequence of experiment results, plan the next experiment, modify train.py, run it, evaluate — merge if improved, discard if not

> "I now have AI Agents iterating on nanochat automatically, so I'll just leave this running for a while, go relax a bit and enjoy the feeling of post-agi :)" — [@karpathy](https://x.com/karpathy/status/2029701092347630069)

## Key Results

- **Round 1 (2 days, ~700 experiments)**: GPT-2 training time **2.02h → 1.80h** (11% reduction)
- All ~20 agent-discovered improvements were **additive** and **transferred** to larger models (depth=24)
- Specific bugs/optimizations the agent found:
  - Missing scaler multiplier on QKnorm → attention was too diffuse
  - No regularization applied to Value Embeddings
  - Banded attention window too conservative (tuning oversight)
  - Misconfigured AdamW betas
  - Optimized weight decay schedule and network initialization

> "Seeing the agent do this entire workflow end-to-end and all by itself as it worked through approx. 700 changes autonomously is wild." — [@karpathy](https://x.com/karpathy/status/2031135152349524125)

## Achievements and Limitations

**Key achievements**
- **Immediately usable open-source framework** enabling anyone to reproduce autonomous ML research (active community forks for MacOS, Windows, AMD)
- **11% improvement on first attempt** on a project already well-tuned by a 20-year veteran — discovering real bugs the expert had missed
- Agent discoveries **transfer to larger models** — optimizations found on small models have practical value at scale

**Limitations**
- Current constraint of **single file (train.py) + single GPU** — far from the complexity of frontier model training (multi-file, distributed)
- Agent stays at the level of **hyperparameter tuning and bug fixing**, not "novel, ground-breaking research"
- Fixed 5-minute time budget differs significantly from real large-scale training feedback loops
- 🤖 Multi-agent collaboration and large-scale scaling are only mentioned, not implemented — the claim that this is "just engineering" remains unproven
